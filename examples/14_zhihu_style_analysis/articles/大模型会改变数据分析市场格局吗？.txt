一个投资了 gitlab 、Character.ai 的投资人说，有人在做下一代AI

数据栈

嘛，

http://

modal.com

的创始人说，需要重新设计新的数据栈

那时候是 2023.4.23，然后 7 月份，

openai

发布了 code interpreter插件。为大模型内置了一个 python 解释器，能够运行 python 代码，进行数据分析和

可视化

。

操作的方式是这样：上传一个

csv文件

，或者一堆csv文件的压缩包。然后可以采用对话方式询问数据问题，或者做一定计算，形成可视化的图标。我找了一个别人的例子：

从上面这个过程来看，对于非技术背景的人来说，非常容易理解和操作。简化了

数据分析

长长的链路，能够让业务用户直接参与数据分析过程，有利于快速决策

这会带来什么改变呢？

当然，前提是 code interpreter 整个过程能够达到一个业务用户基本可用，错误率不大的情况下。从目前的一些分析来看，好像 coder interpreter 代表了 GPT-4.5 的能力，还是很强的

数据分析市场从操作方法来看，有两类：数据工程、数据科学。数据工程是面向数据，

处理数据

。就是把数据按照一定的

数据处理

方法论来处理好给业务部门使用。例如按照 schema、

数据血缘

、数据集市等等。而数据科学，则是基于数据，面向决策，使用统计学、

数学方法

在数据上做演算探索，挖掘数据中的业务价值

从目前 code interpreter 的操作流程来看，上传文件，描述自己想做什么，甚至找一些

算法

来让模式尝试。这个过程非常像

数据科学家

所做的事。当然，数据科学家还有跟多的价值，利用科学方式进行分析探索，或者创造更好的

科学分析方法

，有利于数据更加凸显价值

所以，code interpreter 可以为数据科学家和业务使用者赋能。基础的已经

固定化

的范例，业务使用者可以直接使用 code interpreter 来探索数据。而数据科学家创造更多的科学分析方法。

数据工程师

仍然是需要对数据建模，提炼清洗数据供

code interpreter

使用

但是，code interpreter 边界会不会扩大？从“好事者”的角度，code interpreter 会进攻到数据科学家和数据工程师的原有位置吗？

因为

大模型

是一种训练机制，如果能够从基础数据开始，也就是源头数据开始，建立训练和探索反馈机制。大模型应该能够建立对数据的全新探索分析能力。例如大模型对互联网文本数据的处理，和 python 代码的分析处理，已经显示了极高的理解能力。所以，不排除随着大模型对数据分析的能力提高，边界扩大，可以直接做数据工程师和数据科学家的事

这样就是上面 twitter 里

http://

modal.com

创始人说的，需要重新设计数据栈

但是还有一块，就是数据基础设施。如果大模型从源数据开始，但是数据的分析除了理解之外，还有一个性能问题。在超大数据量的情况下，光靠一个 python 解析器肯定解决不了这个问题。这就带来了新的可能，是不是 code 解析器也可以高性能？或者有更强的架构来适应 code interpreter

code interpreter 的使用量还不知道是什么情况，市场和用户是否对这个操作方法感觉兴奋，愿意尝试还不得而知。但是它提供了一种新的可能，非常期待有创业公司能够在此基础上做出更进一步的产品来
