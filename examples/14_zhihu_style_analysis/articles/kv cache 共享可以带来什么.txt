kv cache 共享可以带来什么

最近

xinference

增加了一个新的能力：

kv cache 共享

基本实现的机制，就是在多副本的 vllm 推理的时候增加 kv cache 的共享。实测结果，第二次查询，如果有重复内容，速度可以提高一倍

在 SGLang 0.4 版本，也出现了一个新能力。叫做 SGLang Cache-Aware Router (缓存感知路由)

基本原理是通过缓存感知路由来匹配 worker 节点的 prefix kv cache ，并且选择匹配率最高的节点去执行任务

在 2024 年底，在

Ray Summit

上，vLLM 的李卓翰和杜昆泰做了一个演讲：

The State of  vLLM 2024

。描述了 vLLM 在 2024 年的工作和 2025 计划

除了模型和硬件的适配之外，还包含了对 Cuda kernel 的优化、量化方法、Chunked Prefill、动态Speculative Decoding、Hash Prefix Caching、多 LoRA Serving、结构化输出、分布式多机推理(单模型)、流水线并行、CPU 卸载

其中有几个重要的针对 kv cache 的优化

主要的改变也是通过识别 prefix cache在并行采样和抢占中可以重用的 kv cache

还有一个有趣的项目叫做

LMCache

，就是类似在 LLM 的 Redis 的概念，这个和 xinference 的思路很像

就是通过共享 vLLM 节点的 kv cache，甚至加载到本地硬盘或者其他的服务器，来实现推理加速。按照 LMCache 的测试结果，可以有 5.7 倍的加速效果

总结一下：KV Cache 共享通过让多个请求复用已计算的注意力缓存，可以减少重复计算、节省显存开销，从而提升推理性能和吞吐量，特别适合处理具有相似输入的多个请求场景。

另外， KV Cache的有效管理，还有助于未来 Prefill 和 Decode 分离的实现。 xinference 做了一个测试：

Image.jpeg

采用 Prefill decode 分离，用 N卡+国产卡，可以相对于纯国产卡，上下文 32k，可以有 3 倍的性能提升。这个还是在普通网卡环境下的测试结果。主要原因还是在算力受限的情况下，传输的开销远小于计算开销

关于 PD 分离的一些分析，后续我们再展开来说
