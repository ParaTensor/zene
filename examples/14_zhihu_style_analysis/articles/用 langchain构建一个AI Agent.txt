最近 OpenAI 联合创始人

Karpathy

分享了对 AI Agent 的看法。认为 AI Agent 是他目前认为最兴奋的事

Open AI 的 Lilian Weng 在个人博客发表了一篇文章：

LLM Powered Autonomous Agents

描述了 Agent 系统的全貌：

看到这个结构，感觉抽象的相当好。Agent 是中心，理解为可以处理一些复杂操作的“代理”服务；Tools 是提供给 Agent 的工具，例如计算、搜索网络、代码执行等等；Memory 是数据库或者其他存储上保存交流的历史记录，防止交流过程中遗忘之前的信息；Action 没有多解释；Planning 就是 LLM 的一些能力，包括反射、目标分解、反思、链式思考。

基于这个结构，Agent 就可以完成一些非常有趣的工作。文章提了一个发现有机药物的过程，通过 langchain 实现了 ReAct 和

MRKL

的过程(LLM 的常用推理方法)，通过

CoT

(思维链)和工具结合，正确性好于 GPT-4

其实可以理解，因为 GPT-4 在专业领域不可能有非常准确的信息。而且专业领域的研究探索过程，是有很多现有的工具沉淀了中间步骤。所以这些结合起来，再加上 LLM 的推理规划能力，效果应该值得期待

在 langchain 里面，有 Agents 这个模块。agent 类型有

多个

，包括：Zero-shot ReAct、Conversational 以及 Plan-and-execute等

其中有好几个是

ReAct

的 agent，ReAct 就是所谓 Reasoning + Acting 模式。就是把复杂问题拆开，有些缺少的内容通过工具从外部获取。然后补充到问题回答里。

上面是一个例子。问题是：除了Apple Remote之外，还有什么设备可以控制 Apple Remote 最初设计用于交互的程序吗？

然后 ReAct 就会拆开，第一个 Thought： 我需要查找 Apple Remote原生的交互程序；然后进行一个 Action，去 google 找了了这个问题答案。然后进行下一个 Thought：Apple remote 是一个控制媒体的程序。我需要查找其他可以控制媒体中心的程序，然后继续下一个 Action：找不到；然后继续 Thought：找其他相关的；然后找到了。把找到的信息填入到最后一个 Thought，得到了答案

所以 ReAct 就是通过外部工具和反复查找匹配得到最终答案的过程

这样似乎给我们打开了一个新的解决问题的路径。利用 LLM 的拆解问题的能力，在中间过程投喂信息。人工参与确认是否正确，直至找到人类认可的答案

那么 langchain 是怎么实现的呢？

Langchain 设置了两种模式：

一种叫 Action Agents，就是下一步的动作由上一步的输出决定

一种叫 Plan-and-execute agents，就是计划好所有的步骤，然后顺序执行

然后在执行过程中， agent 可以调用一些 tools，来辅助过程。例如 google 搜索、语音识别、文件处理、python执行等等。tools 就是给 agent 配备了一些工具箱

'The current temperature in Pomfret is around 70F with partly cloudy skies and winds W at 5 to 10 mph. The humidity is 41%.'

上面代码逻辑还是比较清楚。设置工具，设置 chain。Chain 就是 langchain 的主要概念，是一组执行的封装。运行之后，会出现： Thought --> Action --> Observation 的模式。这就是上面所说的 "ReAct" 模式

在执行

run

之后，langchain 会按照模型来顺序执行。这里是默认 AI Agent 执行模型。就是根据返回结果执行下一步。

这里就会有一个疑问，为什么提交到 LLM 之后，可以返回 Thought-->Action-->Observation 的模式。奥秘还是在 提示词(prompt)模板上。

在使用

agent=AgentType.CONVERSATIONAL_REACT_DESCRIPTION

之后，prompt 就会被格式化成如下这样：

"Have a conversation with a human, answering the following questions as best you can. You have access to the following tools: Search: useful for when you need to answer questions about current events Use the following format: Question: the input question you must answer Thought: you should always think about what to do Action: the action to take, should be one of [Search] Action Input: the input to the action Observation: the result of the action ... (this Thought/Action/Action Input/Observation can repeat N times) Thought: I now know the final answer Final Answer: the final answer to the original input question Begin!"

在提示词模板里面，说明了需要按照 Search、Question、Thought、Action 这种模式来回答，还告诉 LLM ，可以使用 Search 这个工具

然后 LLM 按照要求回答了，langchain 就可以按照格式返回获取对应的 Thought。然后可以提供新的工具或者人的交互输入，形成新的提示词再继续提交给 LLM，直至获得最终结果

这个做法打开了新的一扇窗，流行的 AutoGPT、BabyAGI 也都是类似的机制

当然，AI Agent 还会继续发展，我们可以慢慢尝试把一些场景用这种模式来套入，看看是不是会有新奇的发现
