最近刷

Cloudflare

博客的时候，我点开这篇《How we rebuilt Next.js with AI in one week》，一看发布时间 2026 年 2 月 24 日，直接愣住了。一个工程师，花了 1100 美元的 token，用 AI 硬生生在一周内把 Next.js 给重做了一遍，还做成了一个叫

ViNext

的 drop-in 替代品，直接一条命令就能部署到 Cloudflare Workers 上。

这不是什么概念验证，也不是玩具项目，而是真真正正能跑生产的东西。

我自己这些年用 Next.js 做过不少项目，最烦的就是部署到 Workers 上的那些弯弯绕绕。以前要么用 OpenNext 这种“逆向工程”的方案，脆弱得要死，版本一升级就寄；要么就得自己改 build 输出、适配各种 platform API，折腾得头大。现在 Cloudflare 直接把 Next.js 的整个 API 表面（

App Router

、Pages Router、

RSC

、

Server Actions

、缓存、中间件……）全部在

Vite

上重新实现了一遍，做成了一个 Vite 插件。

意思就是，你原来的 `app/`、`pages/` 目录结构完全不用动，`vinext dev`、`vinext build`、`vinext deploy` 三连，搞定。

更狠的是性能表现。他们拿一个 33 个路由的 App Router 项目做了基准测试：

- Next.js 16.1.6（Turbopack）：构建 7.38 秒，客户端 bundle 168.9 KB（gzipped）

- ViNext（Vite 7 / Rollup）：4.64 秒，74.0 KB

- ViNext（Vite 8 / Rolldown）：1.67 秒，72.9 KB

构建快 4.4 倍，bundle 小 57%。我看完数据第一反应就是：这不就是我梦寐以求的 Next.js 吗？

最让我感慨的是他们的做法。他们没有去 fork Next.js，也没有造一个新框架，而是直接拿 Vite 这个已经非常成熟的构建工具当底座，用 AI 把中间所有的 Next.js API 兼容层全部写了出来。整个过程没有中间框架，没有复杂的适配层，就是“API 契约 + 构建工具 + AI”三件套，AI 把中间那块全干了。

他们还做了个实验性的

Traffic-aware Pre-Rendering

（TPR）：部署的时候直接查 Cloudflare 的流量数据，只把真正热点的页面预渲染出来。比如一个 12847 个路径的站点，只预渲染了 184 个高流量路径，就覆盖了 90% 的流量，构建时间直接起飞。

现在 ViNext 已经开源在 GitHub（cloudflare/vinext），`npm install vinext` 就能用，还带了完整的迁移 Agent Skill，Claude Code、Cursor、Codex 都能直接帮你把老项目转过去。

我刷完整篇文章，心里就一个感觉：AI 写代码的时代真的来了，而且来得比我们想象的快得多。一个工程师 + 1100 美元 token，就能把一个行业标准框架重做一遍，还做得更快、更小、更适合边缘部署。这意味着什么？意味着以后我们选框架的时候，不再是被大厂绑死，而是可以真正挑最适合自己部署平台的那个底座。

Cloudflare 这波操作，不只是给 Next.js 用户多了一个选择，更是给整个前端生态甩出来一个信号：用 AI 重建框架的门槛，已经低到让人头皮发麻了。

对于自己很熟悉的领域，复制一个新的真的很快。但是软件本身有运行价值，也就是 Next.js 之前的坑，大家都踩过了。用 ViNext 你愿意切换吗？
