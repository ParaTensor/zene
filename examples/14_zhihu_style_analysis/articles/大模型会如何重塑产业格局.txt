大模型

从 2023 年初到年底，轰轰烈烈一年发展。从之前的惊叹甚至恐惧，到落地后的一丝丝希望。随着能力的逐步提升，新的希望又产生

但是随着全社会全行业的尝鲜者抱着对大模型的热情和积极尝试，大模型已经在一些方面改变了行业的一些方方面面。这里试图模式一下改变的迹象和探讨未来可能的发展

大模型作为一个新的事物，跳入了大众视野。从行业来看，怎么认识大模型的角色是一个蛮有趣的过程。

OpenAI

研究员认为大模型是操作系统（OS），

微软

研究员认为大模型可以作为通用计算机。使用者会认为，大模型是一个新型的搜索引擎；或者是数据分析平台，代理服务等等

究其原因，因为这次大模型具备了"推理“的能力。能够理解和处理复杂的数学、逻辑和因果关系等。如果从这个角度来说，大模型已经超越了操作系统

操作系统和应用交互，把应用的操作系统转换成对计算机 CPU 、内存和其他设备的操作需求

但是让大模型来承担任务，操作路径可能变成了这样：

用户向大模型提交了任务需求，如果是简单任务，大模型就直接返回数据；如果需要调用他能力，就通过 Agent 在操作系统上运行任务，返回给大模型

这里面有一个非常大的变化，就是大模型是运行在

GPU

上的。对 GPU 的管理是通过英伟达

CUDA

或者 AMD 的

RoCM

来调度的，而且 GPU 有自己的内存和并行调度机制

随着 AI PC的发展，GPU 在计算机里面的占比越来越大。例如 Mac M3 Max，CPU核数为 16，GPU 核数为 40 。当然，核数并不代表计算能力。但是如果越来越多的工作通过大模型来调度的话，GPU 在计算机的比重越来越大会是一个大的趋势。

Canalys 预测，到 2027年，60% 的 PC 将具备 AI 能力

那这个时候，就面临一个比较大的变化，操作系统和 CPU 的作用越来越低。 会出现新的 AI 操作系统吗？原有的计算软硬件体系会有大的改变吗？

这个问题可能要等待未来的发展情况来验证，从目前来看。 Apple 和 Intel 已经非常明显在PC上加强 AI 能力。模型量化、 CPU 和 GPU 混合调度计算模型等都在飞快发展。我们应该很快能看到这方面的变化

应用领域的变化在 2023 年主要集中在 RAG(外挂知识库，增强搜索) 和 Agent，RAG 通过向量召回的方式帮助大模型补充外部知识，增强搜索结果准确度，得到了较好的效果。 Agent 在下半年蓬勃发展，应该会在 2024 年会有更大的发展

RAG 实际上是针对企业大量的非结构化数据进行处理。数据处理有一条传统的处理线路：原始数据->ETL->数据库->数据湖。 RAG 的出现，在数据分析链路上产生了改变：

Databrick 的 RAG 应用在 Retrivers 环节，就同时支持了结构化数据和非结构化数据

从用户角度，如果能够用自然语言直接处理数据。非技术类客户未来可能会更接受这个链路，而不是用 SQL 或者 Python 来做数据处理。这也是未来发展的一个看点

大模型目前还存在准确度、幻觉等问题，如果在 2024 年在基础模型能力进一步提高的基础上。用户和企业都会积极采用大模型落地解决方案，对行业的改变就会更快。这块仅仅做个抛砖引玉，期待能够看到更多的变化
