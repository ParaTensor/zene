最近

Claude Code

和

Claude Opus

4.5 又把整个时间线刷得满满当当，我每天刷着那些“下午搭了个项目就月入五位数”的故事，心里其实挺有感触的。不是说这些故事不真实，也不是在否认 AI 的强大，而是我越来越觉得，这波热潮背后藏着一个很现实的问题：代码的生成成本已经快要归零，可真正能跑得稳、用得久的软件，成本却一点都没降下来。

我自己试过各种低代码平台、

Replit

、

Lovable

之类的工具，确实能十分钟就搭出一个能用的东西。可当我真正把 Claude Code 用熟了以后，发现一个很有意思的变化：以前那些靠可视化平台快速出活的 makers，现在很多都开始转向 CLI-first 的工作流了。为什么呢？因为把交互搬到终端里，中间那层抽象突然就薄了，你不再是被平台牵着鼻子走，而是真正握住了方向盘。我自己最近用它做的几个小工具，全是纯本地、零配置、五分钟就能跑起来的那种，用完就关，特别符合“一次性解决眼前问题”的感觉。

这其实是一个挺大的信号。软件正在从过去那种卖给大众的“商品”，慢慢变成每个人都能给自己定制的“个人工具”。以前遇到一个具体痛点，我们通常会花好几个小时去搜，看看有没有现成的

SaaS

能覆盖八成需求。现在不一样了，直接打开 CLI 或者语音界面，把需求一说，AI 就给你生成一个只属于你自己的方案。订阅怎么记账完全按自己的习惯，Chrome 扩展只管那个极小众的数据录入场景，健身打卡的界面和流程也彻底按自己的审美来……这种变化，让我真正感觉到软件的民主化落地了，不再是口号，而是实实在在每个人都能上手的事。

说实话，这也让我想起自己当年用 Excel 的日子。那时候打开一张表格，不是为了建一个永不删除的数据库，而是当草稿纸用，算完当天的数据，保存一下，第二天直接新建一张。Claude Code 现在对我来说就是程序员的 Excel，它强大、灵活，专门用来思考问题、快速验证想法、解决眼前的麻烦，而不是用来建什么永久的帝国。很多新做的工具压根不打算活很久，大家越来越习惯做一个精准解决今天这个问题的玩意儿，干完就扔。这和传统 SaaS 完全是两条路，SaaS 要考虑留存、付费、规模化，所以天生就得做得很重，而现在的个人工具只追求即时性和控制权，临时反而成了它的核心优势。

但越是这样，我越来越觉得：

LLM

把写代码的成本打到了接近零，可真正把软件做好的成本，几乎没怎么变。我见过太多周末项目，

CRUD

加几个第三方 API 包装一下，在 Twitter 上演示的时候特别惊艳，可真正上线后，改个 CSV 格式就崩了，网站改个 DOM 结构就挂了，用户一提离线同步就直接用不了。这些问题从来都不是代码写得不好，而是

软件工程

本身的问题。真正的成本不在第一行代码，而在后面那些边缘情况的处理、长期的维护、不断积累的 UX 债务、数据同步的复杂性，还有

整个架构能不能扛住真实世界的摩擦

。

AI 看起来那么厉害，正是因为它把复杂性都藏起来了，可作为工程师，我们的活儿恰恰是把这些复杂性管好，而不是假装它不存在。工具确实变了，但对严谨性的要求，反而比以前更高了。那些喊着“软件工程要完蛋了”的声音，我觉得都没抓住重点。我们不是在见证这个职业的消失，而是在进入它的下一个阶段。工程师的价值，正在从“语法层面怎么写”，慢慢转向“系统层面到底要做什么、为什么这么做”。你得知道怎么搭一个能长期活下去的系统，得清楚什么时候该加速率限制、怎么管

分布式缓存

、哪些地方绝对不能放环境变量。

当然，事情还有另一面。入门门槛消失的同时，噪音也达到了历史新高。我的信息流里到处是各种“AI 创业者”说自己一个下午的项目就做到五位数 MRR，很多时候这些数字听起来都挺可疑的。没有现成的分发渠道、没有明显的护城河，一个周末项目就月入上万，通常更多是为了吸睛，而不是真实的商业图景。真正厉害的那些人，靠的也不是技术本身，而是精准抓住了流量红利、对用户痛点极强的直觉，以及把简单功能包装成刚需的能力。现在生成代码已经不是瓶颈了，真正的瓶颈变成了分发能力和你自己的判断力、品味。

我自己现在用 Claude Code 的方式也变了很多。我还是会让它帮我生成大部分代码，但剩下的那部分，我会像 review 老同事的 PR 一样，一行一行看过去。因为我知道，AI 可以让我跑得更快，但最后对整个系统负责的，只能是我自己。这些工具确实能帮我们省掉很多重复劳动，尤其适合领域专家快速验证想法、内部团队搭一次性工具、超级用户把手动流程自动化，或者像我这样更看重解决方案所有权的工程师。但它们永远代替不了多年的经验，和对整个问题空间的理解。

归根结底，这波 AI 热潮并没有把我们带进 SaaS 的黄金时代，而是把我们推向了一个

个人软件的时代

。你可以用二十块钱、几个小时的空闲时间，再加一点耐心，就做出一个真正能解决自己问题的应用。想法和最终成果之间的距离，比历史上任何时候都要短。可也正因为这样，把事情真正做对的门槛，反而更高了。工具在进化，但判断力、品味、责任感、架构思维，这些东西从来都是稀缺的。

你最近在用 Claude Code 或者 Cursor 做什么有趣的小工具吗？可以一起聊聊“创作”的体会
