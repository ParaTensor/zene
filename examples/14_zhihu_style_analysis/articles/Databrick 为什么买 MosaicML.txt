在今年的 Data AI Summit 上，Databrick 联合创始人 Reynold Xin 这样说:

这里面最后一句比较有意思， Python 是 AI 时代的语言，而 Databrick 核心架构是建立在 Java 之上的。但是 Databrick 的联合创始人说 Python 是新的“

byte code

“，我理解就是AI 时代的基础语言

所以是不是可以这么理解，LLM 出现之后，大数据时代占据

数据处理

核心位置的 Databrick 也频频提出了一些针对 LLM 的方案。但是好像没有突破到 LLM 生态链的核心。Mosaic 作为新的 LLM 的训练部署模型平台，成功了获得很多客户。所以 Databrick 收购 MosaciML就比较合理，补齐自身在 LLM 的不足

这就带来另外一个问题，LLM 整个流程大家为什么不用 Databrick Spark？

核心还是那张图上的第三句：Python 是 AI 时代的基础语言，而 Databrick 内核是 Java

最近看到

openai

的文档示例代码：

你会发现，在 AI

处理数据

都喜欢用 pandas，然后用

sklearn

做一些基础特征分析。然后丢入大模型，获得进一步结果，循环往复。

在机器学习领域， Python 的库数以百计，而且每周都会有新的库产生。模型和数据处理又交织在一起，如果在这个过程中使用 Spark来处理，然后再把数据传递给 Python 来计算，势必会带来极大的损耗和开发效率的降低

所以，Databrick 很早就推出 ML 产品，但是在这个领域，一直就无法获得巨大进展

MosaicML 有一个处理数据的产品叫 Streaming

这个产品的开发语言比例是： python 99.6%

所以，可以了解 Python Native 在 LLM 生态的重要性了
