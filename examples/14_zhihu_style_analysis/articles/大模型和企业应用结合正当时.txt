今天 Llama2 的发布又给大模型推上了一个小高潮。媲美 GPT-3.5、开源可以商用、上下文长度4K，感觉真香。实际上，这也给了大模型私有化的一个非常可行的路径。具体 Llama2 的内容可以看一下这篇回答：

然后这一次发布还有一个特点，微软竟然参与其中，是 Llama2 的云合作伙伴。同时，微软 Office 365 Copilot 定价每用户每月30美金

近期还参加了几个活动，一个是

数据处理

领域，一个是 SAAS 服务领域，还有就是大模型的几个会。有一点

平行世界

的感觉。数据处理领域希望引入大模型来做

text2sql

，但是大模型的不准确性又不能应用在生产服务商。saas 服务大量希望采用“机器人”或者 copilot 助理的形式来为客户提供服务，但是如何和 SAAS 服务本身的特点结合又存在难度。

上面这张图是 A16Z 关于 LLM 应用的分析。如果要应用大模型，在数据处理、

流程编排

和应用服务上看上去会有较大的改动

更核心的问题是数据，对于企业或者应用服务上来说，对于把数据提交给大模型也会犹豫。更想的是把大模型能力引入到应用内部，融合之后增强服务能力

所以在这个事上，强大的开源可商用模型就至关重要。所以 Llama 2 的发布就会让大家非常期待，甚至可以说期待已久

另外，阻挡大家使用大模型的一个门槛GPU 的成本太高。这次 Llama 2 的发布还有一个合作伙伴

高通

，甚至提供了基于手机上运行 Llama 2 的方案

而大模型 ggml “量化”技术可以把模型变得很小，可以在单卡，甚至在 mac 上可以跑。这个可以看这一篇：

另外，关于和 LLM 交互的方法也在发生变化，尤其是 langchain 和 ChatGPT code interpreter 这种新的交互方式也给企业应用带来了新的想象力。

在这篇文章里，还提出了一个

任务驱动的 agent 模式

:

能够根据已完成的结果生成新任务以及实时确定任务的优先级来完成任务

这就带来一个非常有趣的问题，未来的企业智能化应用会怎么发展？

企业目前的应用是基于数据、规则化流程和web交互为主要组件组成的。典型的类似下面这样：

企业按照自己运作的方式，为销售、客户服务、财务等提供操作企业内部数据的能力，然后设置一些流程来做管控。那如果改编成以智能 agent 的形式，每个 agent 处理自己那一部分，公司成员按照自己的需要给 agent 提要求，获得返回。那原有的

企业架构

是不是就会变成以 agent 为中心了。为 agent 提供数据、特定的模型、权限管控等等

当然，这个过程不会一帆风顺。我们需要考虑的是，是不是来尝试和推进这个模式。这次 Llama2 给了我们很好的基础来尝试，后面一段时间应该会出现很多变化，让我们拭目以待
