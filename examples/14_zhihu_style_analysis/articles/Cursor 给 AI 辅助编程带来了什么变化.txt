最近

Cursor

辅助编程火了。从

Andreessen Horowitz

、

Jeff Dean

、

John Schulman

、

Noam Brown

以及 Stripe 和

Github

的创始人那里筹集了 6000 万美元。而且得到了硅谷大佬普遍的好评

Cursor 自己定义为 "AI Code Editor"，也就是 AI 代码编辑器。长这个样子

你没有看错，这个就是

vs code

编辑器。然后 cursor 增加了一些能力。例如左边这个

AI Pane

的界面，是它最主要的操作界面。还有一个

cursor composer

界面，是 AI Pane 更复杂的版本，可以先看一眼：

Composer 提供了前后修改的代码比对的能力

在 AI Pane ，有一个最大的特点。也可以说是 Cusor 的核心特点，就是可以上下文。

例如上面，我可以用 @ 这个标记，来选择这段提示词的范围。例如从哪个网址寻找，相关的代码文件甚至代码片段在哪里

这样 Cursor 就可以把生成的上下文比较准确的提供给模型，以此来达到更准确的生成。

当然，Cursor 还有更多的编程交互体验的创新。但是主要产生的化学反应，应该是

Claude

的模型+精确制定的上下文

我花了一天时间，写了一个小应用。就是类似

Notion

的文本编辑，有几个体会：

操作非常高频。因为很难一次描述清楚需求，需要反复修改尝试。就对应软件开发中非常频繁的debug

生成的代码可能更复杂，如果是人来写，肯定不会这么写。但是好像这样写也挺好的，所谓的抽象、简化好像也没必要。传统的软件简洁之道似乎没必要遵循

效率提高可能有 3-5 倍。而且生成的代码出错较少

一开始会非常不适应，但一旦体会到快速开发带来的乐趣之后就停不下来。大部分时间不用去扣代码的细节，主要是提示词交互

最近和几个软件公司的资深开发者交流，Cursor 和

Github copilot

这类辅助编程工具给开发人员带来了巨大的效率提升。对产业可能带来深远的影响：

较强的小团队可能能够开发出和大公司产品匹敌的商业化软件

纯粹的软件外包公司面临“死亡之吻”

用《创新者的窘境》作者

克莱•克里斯坦森

的话来说，这些10倍的改变叫作“颠覆性创新”（disruptive innovations）；用

安迪：格鲁夫

的话来说，叫作“战略拐点”（strategic inflection points）。不论采用哪种术语，战略家的角色都是辨识这些转变，并且设计出战略来应对这些转型期。如果事后才意识到这些转变，10倍的改变可能就是死亡之吻；

-- 来自《战略思维》

现在 AI 辅助工具对效率提升可能是在2-3 倍。如果工具发展更好，开发人员能够更熟悉使用工具，效率提升可能就到5-8 倍。离10 倍速是可以预期的

随着 zed.dev 和 v0.dev 加入这场创新，加上基础模型大约一个季度一次的大的能力提升。可以期待 10 倍速开发的真正到来了
